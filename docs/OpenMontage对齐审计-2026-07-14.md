# OpenMontage 对齐审计（2026-07-14）

## 结论

当前项目完成了生产主链路，但只吸收了 OpenMontage 的一部分工程思想。不能宣称功能已经全部对齐。

## 已对齐

- 声明式阶段流水线与工具注册表。
- 脚本、关键帧人工闸门，以及归档后重新打开人工节点。
- 单镜失败隔离、重试、版本记录和供应商错误记录。
- FFmpeg 统一输出、ffprobe 媒体检查、失败禁止归档。
- 项目列表、节点状态、运行报告、成本观察模式。
- 参考视频只提取结构与节奏，产品事实和素材独立约束。

## 部分对齐

- Living storyboard：已有分镜编辑和逐镜运行，但没有 take A/B 对比、质量分和逐镜批准状态。
- Provider decision log：已有调用模型和错误元数据，但没有七维供应商选择评分。
- Delivery review：已有分辨率、时长、音轨和抽帧人工复核，但没有音量、静音、字幕越界与承诺一致性检查。
- Run replay：已有事件和时间戳，但没有可拖动回放时间线。
- Remotion/HyperFrames：当前只采用 FFmpeg 合成，尚未引入字幕、动效模板和品牌版式渲染层。

## 尚未完成

- 逐镜多 take 管理、contact sheet 和选择后再合成。
- 自动字幕、逐词时间码、字幕安全区与可读性 QA。
- 旁白/TTS、音乐、响度归一和音频峰值检查。
- 运行回放时间线与阶段耗时瀑布图。
- 30 秒恒温杯真实回归、录屏和 Claude Code 验收包。
- TikTok 授权 API/OAuth 通道；当前只有 oEmbed 与人工导入。

## 不直接照搬

- 不复制 OpenMontage 全部 12 条通用管线；本项目保持海外母婴硬件短视频这一条受控主链路。
- 不在产品可见镜头使用通用图库替代公司素材。
- 不在当前阶段引入多个视频供应商和复杂成本路由，先稳定 Seedance 实际可用率。
- 不恢复反风控抓取或未经授权的视频下载能力。

## 后续顺序

1. Backlot 风格制作看板：节点导航、待处理状态、逐镜 take 与批准。
2. 字幕和音频生产层：旁白、逐词字幕、响度与安全区 QA。
3. Remotion 模板层：品牌字幕、CTA、产品参数和结尾卡；FFmpeg 继续承担标准化与封装。
4. 30 秒真实回归：先 1 条，人工复核通过后最多扩到 3 条。
5. 形成 Claude Code 验收说明、录屏和交付包。
