# 内网部署与数据持久化

## 存储架构

系统采用混合持久化：

- SQLite `db/agentflow.db`：项目、任务、状态、成本和事件。
- `data/01_素材库`：产品资料、TikTok 视频、封面、转写和拆解结果。
- `data/runs/{project_id}`：脚本、分镜、Take、合成视频、质检和交付产物。
- `data/05_反馈库`：人工反馈。

视频和图片不写入 SQLite BLOB。内网服务器必须把 `db/` 和 `data/` 放入持久磁盘并纳入备份；只部署代码目录会导致重启或更新后丢失产物。

## 最小部署检查

1. Python、FFmpeg、yt-dlp、Playwright Chromium 已安装。
2. `.env.local` 或服务环境变量已配置，密钥不进入 Git。
3. 服务账号对 `db/`、`data/` 和 `logs/` 有读写权限。
4. `GET /healthz` 返回 `ok`。
5. `GET /api/v2/runtime` 中模型和至少一个 TikTok 后端显示可用。
6. 完成一次恒温杯真实链路，最终视频为 720×1280、约 30 秒且可播放。

## 备份建议

每天备份 `db/agentflow.db`、`data/01_素材库` 和已交付项目目录。SQLite 使用 WAL 模式，在线备份应使用 SQLite backup 命令或先执行 checkpoint，避免只复制主数据库而遗漏 WAL 中尚未合并的数据。

## 节点下载

脚本、脚本拆解和分镜可从对应页面下载 JSON；每个视频 Take 可单独下载；合成后可下载 720P 成片；交付页仍提供完整 ZIP 包。
