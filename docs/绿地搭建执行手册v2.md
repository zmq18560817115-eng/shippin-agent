## 1. 最终版七 Agent 职责卡（按团队命名，边界已修正）

**边界原则**：Agent 边界 = 产物契约边界。一份产物只有一个生产者；消费者只认 schema 不认实现。

| # | Agent（中文/id） | 拥有的产物契约 | 核心职责 | 修正说明 |
|---|-----------------|---------------|---------|---------|
| 1 | 抓取 `collector` | `material_meta` / `library_index` | 三通道采集（API→浏览器→人工投递）+ 去重入库 + 缩略图 | 吸收原"素材入库"职责；**独立于主流水线运行**，采集挂了不影响任何在制项目 |
| 2 | 拆解分析 `analysis` | `analysis_report` | ASR 口播稿 + 场景关键帧抽样 + 结构拆解（钩子/节奏/卖点序） | 指纹缓存：同素材第二次拆解 0 token |
| 3 | 脚本 `script` | `script_copy`（**文案层**） | 五段式结构（钩子/痛点/方案/证明/CTA）、英文台词、时长分配、合规词过滤 | **修正①**：只管"说什么"，不写任何视觉字段 |
| 4 | 分镜头 `storyboard` | `shot_plan`（**视觉层**） | 每镜画面描述、visual_prompt、seedance_prompt、运镜（camera_motion）、footage_type | **修正①**：只管"怎么拍"；改台词不触发重做分镜，改运镜不触发重写台词 |
| 5 | 产品素材 `asset` | `asset_manifest` | 白底主图垫图校验（硬闸门）、角色资产、每镜 hero frame 生成 | 校验不过 = blocked，**任何重试不得绕过** |
| 6 | 成片 `media` | `shot_report` / `render_report` | 两种任务型：`shot_gen`（单镜生成，镜头级重试）与 `compose`（拼接+增强+转码） | **修正②**：一个 Agent、两种任务粒度；合成失败只重跑合成，绝不触发重新生成 |
| 7 | 审核反馈 `review` | `review_report` / `qa_report` / `publish_archive` | **双位审核**：位一=脚本评审（出片前，LLM 7 维打分，不合格带意见退回 script ≤2 次）；位二=成片质检（ffprobe 机检 + 人工验收）。另辖归档四库 + 反馈闭环回流 | **修正③**：审核是能力不是工位，前移一份到脚本闸门，把问题拦在烧钱之前 |

```
主流水线（viral-imitate v2）：
analysis → script → script_review(review位一) → 闸门①人工确认脚本
  → storyboard → asset(硬闸门+hero frames) → 闸门②人工确认关键帧
  → media:shot_gen ×N（镜头级） → media:compose → final_qa(review位二) → archive_feedback

collector：独立定时/人工触发，只负责往素材库供货，不在主链上
```

## 5. Artifact 契约 v2（九份，核心两份给全量 schema）

| schema | 生产者 | 说明 |
|--------|--------|------|
| material_meta | collector | 沿用 tiktok_collector 字段 |
| library_index | collector | 入库索引 |
| analysis_report | analysis | hook_3s / structure / voiceover_text / pacing[] |
| **script_copy** | script | **文案层，见 5.1** |
| **shot_plan** | storyboard | **视觉层，见 5.2** |
| asset_manifest | asset | seedance_source 必须指向 白底主图.*（语义校验硬规则）+ hero_frames[] |
| shot_report | media | shots[]：每镜 status/path/cost/attempt（镜头级，第一期即到位）|
| render_report | media | ffprobe：duration/resolution/fps/audio_streams |
| review_report / qa_report / publish_archive | review | 评审 7 维分与意见 / 机检结果 / 归档索引 |

### 5.1 script_copy.schema.json（文案层）

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "script_copy",
  "type": "object",
  "required": ["version","project_id","product_id","sections"],
  "properties": {
    "version": {"const": "2.0"},
    "project_id": {"type": "string"},
    "product_id": {"type": "string"},
    "source_link_id": {"type": ["integer","null"]},
    "total_duration_s": {"type": "number", "minimum": 12, "maximum": 60},
    "generator": {"type":"object","properties":{
        "provider":{"type":"string"},"model":{"type":"string"},
        "prompt_version":{"type":"string"}}},
    "sections": {
      "type": "array", "minItems": 3, "maxItems": 8,
      "items": {
        "type": "object",
        "required": ["number","role","timing","voiceover_en"],
        "properties": {
          "number": {"type": "integer", "minimum": 1},
          "role":   {"enum": ["钩子","痛点","方案","证明","行动号召"]},
          "timing": {"type": "string", "pattern": "^\\d+-\\d+s$"},
          "voiceover_en": {"type": "string", "minLength": 2, "maxLength": 220},
          "subtitle_en":  {"type": "string"},
          "selling_points": {"type": "array", "items": {"type": "string"}}
        }
      }
    },
    "feedback_constraints_applied": {"type":"array","items":{"type":"string"}}
  }
}
```

语义规则（artifacts.py）：number 连续从 1 起；voiceover_en 全量过合规词表；timing 区间连续无重叠且总和 = total_duration_s（±1s）。

### 5.2 shot_plan.schema.json（视觉层，number 与 script_copy.sections 一一对应）

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "shot_plan",
  "type": "object",
  "required": ["version","project_id","script_copy_ref","aspect_ratio","shots"],
  "properties": {
    "version": {"const": "2.0"},
    "project_id": {"type": "string"},
    "script_copy_ref": {"type": "string",
        "description": "所依据的 script_copy 产物路径；script_copy 变更后 engine 使本产物失效"},
    "aspect_ratio": {"enum": ["9:16","16:9","1:1"], "default": "9:16"},
    "shots": {
      "type": "array", "minItems": 3, "maxItems": 8,
      "items": {
        "type": "object",
        "required": ["number","visual","footage_type","camera_motion"],
        "properties": {
          "number": {"type": "integer", "minimum": 1},
          "visual": {"type": "string", "minLength": 4},
          "visual_prompt": {"type": "string"},
          "seedance_prompt": {"type": "string"},
          "footage_type": {"enum": ["AI_BROLL","AI_VIDEO","STOCK","SCREEN","NONE"]},
          "camera_motion": {
            "type": "object", "required": ["type"],
            "properties": {
              "type": {"enum": ["dolly_in","dolly_out","pan_left","pan_right",
                                 "static","arc","crash_zoom"]},
              "start_frame_focus": {"type": "string"},
              "end_frame_focus": {"type": "string"},
              "duration_sec": {"type": "number", "minimum": 1, "maximum": 10}
            }
          }
        }
      }
    }
  }
}
```

语义规则：shots[].number 集合 == sections[].number 集合（一镜一段）；footage_type∈{AI_BROLL,AI_VIDEO} ⇒ seedance_prompt 非空且必须包含产品外观约束句（从 knowledge/output-standards 词表校验）。

**联动规则（engine 内置，修正①的收益所在）**：script_copy 在闸门①被微调保存 → 仅当 voiceover/timing 变化的镜号，其 shot_plan 对应镜标记 stale，storyboard Agent 只重生成 stale 镜；反之改运镜不触碰 script_copy。

## 6. 极简闸门工作台（web/，第一期唯一前端）

**定位**：不是第四批三视图，只服务"能跑起来"：10 个用户看项目、过两个闸门、下载成片。纯 HTML+JS（fetch 调 8790 API），无框架无构建——Claude Code 一次生成，你无前端负担。

单页四区：

| 区块 | 内容 | 调用 API |
|------|------|---------|
| 新建项目 | 产品下拉 + 对标链接/link_id 输入 +「开始生产」 | POST /api/v2/pipeline/run |
| 项目列表 | 每行：项目号 · 7 节点状态灯（●queued ◐running ✔done ✖failed ⏸awaiting）· 成本累计 · 操作 | GET /api/v2/pipeline?limit=50 |
| 闸门①脚本确认 | sections 表格（台词/时长可编辑）+ 评审分与意见展示 +「保存并通过」「退回重写」 | PUT artifacts/script_copy + POST gates/approve |
| 闸门②关键帧确认 | 每镜 hero frame 缩略图 + 运镜摘要 +「单镜重生成」「全部确认」 | POST hero/regen(shot) + gates/approve |
| 交付区 | final_qa 通过的项目：成片预览 + zip 下载 + 一句话反馈输入（写 05_反馈库） | GET download + POST feedback |

失败节点点灯可展开 error_json（tool/reason/镜号）+「重试」按钮——这就是失败隔离的用户可见形态。

## 8. viral-imitate.yaml v2 全文（块 5 直接使用）

```yaml
name: viral-imitate
version: "2.0"
description: 爆款跟拍（七Agent版）：拆解 → 文案 → 评审+闸门① → 分镜 → 素材硬闸+关键帧闸门② → 镜头级出片 → 合成 → 质检 → 归档反馈
default_budget_cny: 35.0
budget_mode: observe

defaults: {max_retries: 2, backoff: exp}

stages:
  - name: analysis
    agent: analysis
    consumes: []                              # payload: {link_id, product_id}
    produces: [analysis_report]
    tools: [doubao_analyze]
    skip_if_artifact_exists: analysis_report  # 指纹缓存

  - name: script
    agent: script
    consumes: [analysis_report]
    produces: [script_copy]
    tools: [doubao_script]
    on_validation_fail: retry

  - name: script_review                       # 审核位一（修正③）
    agent: review
    task_type: script_review
    consumes: [script_copy, analysis_report]
    produces: [review_report]
    tools: [doubao_review]
    on_review_fail: requeue_script            # 带意见退回 script，计数 ≤2，超次 needs_review

  - name: script_gate                         # 闸门①
    agent: review
    gate: human
    consumes: [script_copy, review_report]
    gate_ui_hint: 确认/微调文案；保存仅使受影响镜的分镜标记 stale

  - name: storyboard
    agent: storyboard
    consumes: [script_copy]
    produces: [shot_plan]
    tools: [doubao_shotplan]
    on_validation_fail: retry

  - name: asset
    agent: asset
    consumes: [shot_plan]
    produces: [asset_manifest]                # 含 hero_frames
    tools: [hero_frame]
    hard_block_on_fail: true                  # 白底主图硬闸门

  - name: hero_gate                           # 闸门②
    agent: asset
    gate: human
    gate_flag: HERO_FRAME_GATE
    consumes: [asset_manifest]
    gate_ui_hint: 逐镜确认构图；单镜重生成不触发出片

  - name: production                          # 镜头级（修正②，第一期即到位）
    agent: media
    task_type: shot_gen
    fan_out: shot_plan.shots                  # engine 按镜拆 N 任务
    consumes: [shot_plan, asset_manifest]
    produces: [shot_report]
    tools: [seedance_shot]
    max_retries: 2                            # 镜头级，重试成本=单镜
    budget_guard: true

  - name: compose
    agent: media
    task_type: compose
    consumes: [shot_report, script_copy]
    produces: [render_report]
    tools: [ffmpeg_compose]

  - name: final_qa                            # 审核位二
    agent: review
    task_type: final_qa
    consumes: [render_report, script_copy]
    produces: [qa_report]
    qa:
      - ffprobe_duration_within: {tolerance_s: 1.5}
      - has_audio_stream: true
      - resolution_matches_aspect: true

  - name: archive
    agent: review
    task_type: archive
    consumes: [qa_report, render_report]
    produces: [publish_archive]               # 写 03/04/05 库 + 反馈约束
```
